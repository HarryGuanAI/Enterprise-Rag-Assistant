## 1. 当前项目状态

项目已经完成第一版真实 RAG 问答链路：

```text
文档上传/重新入库
-> 文档解析与混合分块
-> DashScope text-embedding-v4 生成向量
-> PostgreSQL + pgvector 检索
-> 严格拒答判断
-> DeepSeek 流式生成回答
-> 前端展示答案、引用来源、检索调试信息
```

当前本地可访问地址：

- 前端页面：http://127.0.0.1:3004
- 后端 API：http://127.0.0.1:8000
- 后端健康检查：http://127.0.0.1:8000/health

## 3. 已完成的重要能力

认证与权限：

- 内置管理员登录
- 游客模式
- 游客每日 2 次问答限制
- IP 级兜底限流
- 管理员可上传文档、重新入库、修改 RAG 设置

文档处理：

- 支持 PDF、DOCX、Markdown、TXT
- 不支持 `.doc`，后续可通过 LibreOffice 转换方案扩展
- 上传后异步解析
- 混合分块：标题/段落优先，过长内容按固定长度和 overlap 切分
- 短章节不会再被 `min_chunk_size` 错误过滤

RAG 链路：

- DashScope Embedding 生成向量
- pgvector cosine distance Top-K 检索
- 严格拒答：Top1 相似度低于阈值时不调用 DeepSeek
- DeepSeek SSE 流式回答
- 保存消息、引用快照和结构化引用
- SSE 事件包含 `status`、`retrieval_debug`、`citations`、`answer_delta`、`done`、`error`

前端展示：

- 单页工作台
- 顶部统计卡片
- 左侧文档列表和重新入库按钮
- 中间聊天问答
- 右侧引用来源和检索调试
- 管理员设置面板
- 回答区支持轻量 Markdown 展示：加粗、短横线列表、数字列表
- 当前页面已改为固定视口布局，输入框固定在聊天面板底部，左中右三栏各自滚动

评测：

## 4. 本地启动方式

进入项目目录：

```powershell
cd D:\03_Work\04_NewWorks\我的AI新工作\enterprise-rag-assistant
```

启动数据库和后端：

```powershell
docker compose up -d postgres backend
```

启动前端生产服务：

```powershell
cd frontend
npm.cmd run start -- -H 0.0.0.0 -p 3004
```

如果 3004 端口被旧进程占用，可以先停止占用进程：

```powershell
$conn = Get-NetTCPConnection -LocalPort 3004 -State Listen -ErrorAction SilentlyContinue | Select-Object -First 1
if ($conn) { Stop-Process -Id $conn.OwningProcess -Force }
```
