## 1. 当前状态

项目已经从基础 Demo 发展为一个可演示的企业知识库 RAG 助手，主链路如下：

```text
文档上传 / 样例文档导入
-> 文档解析与混合分块
-> DashScope text-embedding-v4 生成向量
-> PostgreSQL + pgvector 入库
-> 向量检索 / Hybrid Search / 轻量 Rerank
-> 严格拒答判断
-> DeepSeek deepseek-chat SSE 流式回答
-> 前端展示答案、引用标签、文档预览、检索调试信息
```

## 3. 已完成能力

认证与权限：

- 管理员登录
- 游客模式
- 游客每日 2 次问答限制
- IP 级兜底限流
- 管理员可上传文档、重新入库、修改 RAG 设置

文档处理：

- 支持 PDF、DOCX、Markdown、TXT
- 上传后异步解析、分块、Embedding、入库
- 混合分块：标题/段落优先，过长内容按固定长度和 overlap 切分
- 短章节不会被 `min_chunk_size` 错误过滤
- 新增 `sample_docs/`，包含虚构企业知识库文档，覆盖 HR、报销、休假、IT、产品 FAQ、客服工单、销售合同、数据安全
- 新增 `backend/app/evals/ingest_sample_docs.py`，可批量导入样例知识库

RAG 与检索：

- pgvector cosine distance Top-K 检索
- 默认 `min_similarity=0.60`
- 严格拒答：Top1 相似度低于阈值时不调用 DeepSeek
- Hybrid Search：向量召回 + 中文关键词召回融合
- 轻量 Rerank：基于向量分、关键词覆盖和内容相关性做二次排序，不额外调用模型
- SSE `retrieval_debug` 展示检索模式、候选数量、阈值、Top1、拒答状态
- 引用中保留 `vector_score`、`keyword_score`、`source`

生成与交互：

- DeepSeek SSE 流式回答
- 保存消息、引用快照和结构化引用
- SSE 事件包含 `status`、`retrieval_debug`、`citations`、`answer_delta`、`done`、`error`
- 回答生成中支持手动停止：前端用 `AbortController` 中断 SSE/fetch 流
- 手动停止后保留已生成内容，并追加“已手动停止输出”

前端体验：

- 当前页面结构已基本确定：
  - 左上角 logo + 产品名
  - 顶部中间轻量统计条
  - 左侧窄文档列表

- 左侧文档栏宽约 230px，右侧引用栏宽约 300px，主要空间留给中间问答

- 回答区支持轻量 Markdown 展示：加粗、短横线列表、数字列表
