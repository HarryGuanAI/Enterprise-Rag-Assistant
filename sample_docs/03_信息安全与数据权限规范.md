# 云舟科技信息安全与数据权限规范

版本：2026.05
适用范围：所有接触公司系统、客户数据、研发代码、生产日志和内部文档的人员。

## 1. 数据分级

公司数据分为四级：

| 级别 | 示例 | 处理要求 |
|---|---|---|
| 公开 | 官网新闻、公开白皮书、招聘信息 | 可对外传播，但不得篡改 |
| 内部 | 普通流程文档、培训材料、内部会议纪要 | 仅限公司内部访问 |
| 敏感 | 客户名单、合同金额、员工薪酬、业务指标 | 最小权限、审批访问、禁止外传 |
| 受限 | API Key、生产数据库、个人身份信息、未公开财报 | 强审批、访问留痕、加密存储 |

## 2. 账号和密码

所有员工必须使用个人账号访问系统，不得共用账号。密码长度不少于 12 位，必须包含大小写字母、数字和特殊字符。核心系统必须开启多因素认证。

离职、转岗或外包人员退场时，HR 应在最后工作日 18:00 前通知 IT 关闭企业邮箱、VPN、代码仓库、工单系统和生产环境权限。紧急离职场景下，权限应在 2 小时内冻结。

## 3. API Key 和密钥管理

API Key、数据库密码、JWT Secret、SSH 私钥和云厂商 AccessKey 都属于受限数据。不得写入代码仓库、README、工单评论、即时通讯消息、截图、日志或前端代码。

如果密钥疑似泄露，应立即废弃旧密钥，重新生成新密钥，并检查最近 30 天调用日志。任何人发现密钥泄露都可以直接通知安全负责人，不需要等待上级审批。

生产环境密钥只允许放在服务器环境变量、密钥管理服务或受控 CI/CD Secret 中。测试环境和生产环境必须使用不同密钥。

## 4. 客户数据处理

处理客户数据应遵循最小权限原则，只访问完成工作所必需的数据。未经客户授权和内部审批，不得将客户数据下载到个人电脑、个人网盘、外部 AI 工具或未经批准的 SaaS 平台。

客户数据导出必须记录导出人、导出时间、数据范围、用途、保存位置和预计删除时间。导出的临时文件应在任务完成后 7 天内删除。

## 5. 脱敏要求

测试、培训、截图、问题排查和产品展示中不得使用客户原始数据。手机号应保留前三后四，中间用星号替代；邮箱应隐藏用户名中间部分；身份证号只保留前 3 位和后 4 位；合同金额可按区间展示，不展示精确数字。

生产日志发给研发排查前，应删除 token、cookie、手机号、邮箱、身份证号、银行卡号和客户密钥。截图中如果包含客户名称、合同金额或人员信息，应先打码。

## 6. 外部共享审批

敏感数据对外共享需要部门负责人和安全负责人审批。审批单必须包含共享目的、数据范围、接收方、共享方式、有效期、删除要求和责任人。

禁止通过个人微信、个人邮箱、公开网盘或未备案 IM 工具发送敏感数据。对外共享链接必须设置访问密码、到期时间和下载限制。

## 7. AI 工具使用边界

员工可以使用经批准的 AI 工具辅助写作、代码解释和资料整理，但不得输入客户原始数据、未公开合同、生产日志、API Key、员工薪酬、身份证明或公司未发布战略。

如果需要用 AI 工具处理内部资料，应先完成脱敏，并确认工具不会把输入内容用于公开训练。无法确认时，应默认不能上传。
